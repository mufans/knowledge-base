知识点：智能缓存系统设计

## 核心概念
通过建立缓存机制，避免重复计算和请求，降低API调用次数和Token消耗。

## 技术要点
1. 分层缓存架构（内存缓存、文件缓存、数据库缓存）
2. TTL过期机制（时间失效）
3. 哈希计算变更检测（MD5/SHA256）
4. 增量更新（只处理变化部分）
5. 缓存命中率统计

## 代码示例
```python
# cache-manager.py
import hashlib
import json
from pathlib import Path

class CacheManager:
    def __init__(self, cache_dir, ttl=86400):
        self.cache_dir = Path(cache_dir)
        self.ttl = ttl
        self.cache_dir.mkdir(parents=True, exist_ok=True)

    def _get_cache_path(self, key):
        # 使用MD5哈希作为文件名
        hash_key = hashlib.md5(key.encode()).hexdigest()
        return self.cache_dir / f"{hash_key}.json"

    def get(self, key):
        cache_path = self._get_cache_path(key)
        if not cache_path.exists():
            return None

        # 检查是否过期
        import time
        mtime = cache_path.stat().st_mtime
        if time.time() - mtime > self.ttl:
            return None

        with open(cache_path, 'r') as f:
            return json.load(f)

    def set(self, key, value):
        cache_path = self._get_cache_path(key)
        with open(cache_path, 'w') as f:
            json.dump(value, f)

# 使用示例
cache = CacheManager('~/.openclaw/cache/api', ttl=3600)
data = cache.get('api_key')
if not data:
    data = fetch_from_api()  # 假设的API调用
    cache.set('api_key', data)

实际应用

  1. 定时任务缓存：技术动态推送、新闻推送避免重复获取
  2. API调用缓存：Context7查询、Figma API缓存
  3. 学习资料缓存：论文下载、教程缓存避免重复下载

学习心得

缓存系统的核心是平衡"命中率"和"过期时间"。命中率越高，效果越好；但过期时间太长会导致数据陈旧。需要根据实际场景调整TTL和缓存策略。 ```
