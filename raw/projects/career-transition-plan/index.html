AI Agent 转型计划（2026.04 - 2027.04）

mufans | 90年生 | 合同到期 2027.05 | 每日可用 6-7h

每日时间分配（6小时）

上午  3h  → 深度开发（项目代码）
下午  2h  → 学习 + 写文章（输入→输出）
晚上  1h  → 社区互动 + 复盘

原则：每天必须有可交付产出，不允许"只学不做"的日子。

Phase 1：基础产品期（4月 - 6月，12周）

目标

  • mobile-dev-agent 上线并开源
  • 掘金/知乎累计 12 篇文章
  • GitHub 个人主页有 3 个项目

第1-4周（4月）：单 Agent 跑通

项目开发（每天3h）： - Week 1：LangChain + Structured Output 基础，单 Agent 代码生成跑通 - Week 2：Few-shot prompt 调优，生成代码质量达标 - Week 3：CLI 入口 + 文件输出，能生成可运行的鸿蒙/Android 代码 - Week 4：GitHub 开源，写 README + Demo 截图

文章输出（每周1篇）： 1. 「移动端开发者的 AI Agent 入门：从 0 搭建第一个 Agent」 2. 「LangGraph 实战：用 TypedDict 管理 Agent 状态」 3. 「Structured Output：让 LLM 返回结构化代码的正确姿势」 4. 「踩坑记录：LLM 生成代码的常见问题和解决方案」

第5-8周（5月）：多 Agent 编排

项目开发（每天3h）： - Week 5：拆分需求分析 Agent，LangGraph StateGraph 编排 - Week 6：架构设计 Agent + 代码审查 Agent - Week 7：条件路由（审查不通过→重试）、迭代控制 - Week 8：端到端测试，优化生成质量

文章输出（每周1篇）： 5. 「从单 Agent 到多 Agent：LangGraph 编排实战」 6. 「多 Agent 协作中的状态管理和错误处理」 7. 「让 AI 审查自己生成的代码：代码审查 Agent 设计」 8. 「mobile-dev-agent 开源：一个移动端智能开发助手」

第9-12周（6月）：MCP Server 开发

项目开发（每天3h）： - Week 9：学习 MCP 协议规范，开发第一个 MCP Server（代码搜索） - Week 10：开发第二个 MCP Server（鸿蒙文档查询） - Week 11：接入 OpenClaw 测试，发布到 GitHub - Week 12：mobile-dev-agent 接入 MCP Server

文章输出（每周1篇）： 9. 「MCP 协议详解：AI Agent 的 USB 接口」 10. 「手把手教你开发第一个 MCP Server」 11. 「用 MCP Server 封装你的领域知识」 12. 「半年复盘：移动端开发者转型 AI Agent 的真实经历」

Phase 2：影响力建设期（7月 - 9月，12周）

目标

  • GitHub 累计 200+ stars
  • 掘金/知乎有一定粉丝基础
  • 开始接触外包/咨询机会

每周节奏

周一至周三  → 继续迭代项目（新功能/重构）
周四至周五  → 写文章/录视频（每周1-2篇）
周末        → 社区互动 + 学习新技术

重点动作

  • 给热门 AI 开源项目提 PR（LangGraph、LangChain、Dify）
  • 参加上海 AI Meetup（每月至少1次）
  • 在掘金/知乎回复 AI Agent 相关问题，建立专业形象
  • 尝试接一个小型 AI Agent 外包项目（验证市场需求）

Phase 3：机会转化期（10月 - 2027.03）

目标

  • 确定下一步：全职 AI 岗位 / 独立开发 / 外包团队

10-12月：试水

  • 优化简历，突出 AI Agent 项目经验
  • 在 Boss 直聘/脉脉上投 AI 相关岗位
  • 同时接外包项目，建立收入来源
  • 参加秋招/内推

1-3月：决策

  • 如果拿到满意的 AI 岗位 → 跳槽
  • 如果外包收入稳定 → 考虑独立发展
  • 如果都不理想 → 加大开源和文章力度，再缓冲

关键指标（每周自检）

指标 4月 6月 9月 12月
开源项目数 1 3 5+ 5+
文章数 4 12 20+ 25+
GitHub stars 10 50 200+ 300+
外包/咨询收入 0 0 有 稳定
人脉（社群联系人） 10 30 50+ 80+

风险预案

  1. 项目做不下去 → 降低scope，先做一个最小可用的工具，别追求完美
  2. 文章没人看 → 换平台/换话题，关注数据反馈，写读者想看的
  3. 身体扛不住 → 减到每天4h，质量 > 数量，别把身体搞废了
  4. 合同提前终止 → 立刻启动找工作，用现有作品集投递

每日检查清单

  • [ ] 今天写了多少行代码？
  • [ ] 今天输出了什么（文章/笔记/commit）？
  • [ ] 明天要做什么？有具体任务吗？
  • [ ] 本周目标完成进度如何？
