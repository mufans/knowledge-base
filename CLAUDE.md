# CLAUDE.md — 知识库规则配置

> 知识库的"大脑"，AI操作知识库时必须遵守的规则。

## 目录结构

```
knowledge/
├── CLAUDE.md          ← 本文件（规则配置）
├── index.md           ← 索引表（所有内容的花名册）
├── log.md             ← 操作日志（append-only）
├── raw/               ← 原始资源层（只读不改）
│   ├── inbox/         ← 每日采集的原始内容
│   ├── projects/      ← 项目相关原始文档
│   └── assets/        ← 图片附件
└── wiki/              ← 维基层（AI提炼的结构化知识）
    ├── concepts/      ← 概念页面（如：RAG、Agent、Memory）
    ├── entities/      ← 实体页面（如：某个项目、某个人物）
    ├── sources/       ← 来源摘要（原始资源的精炼版本）
    └── syntheses/     ← 综合分析（跨源对比、趋势分析）
```

## 核心规则

1. **Raw层只读**：AI只能从raw/读取信息，绝不能修改原始文件
2. **Wiki层可写**：AI可以创建、修改、合并wiki/下的文件
3. **每次操作必须更新log.md**：记录日期、操作类型、涉及文件
4. **每次创建新wiki页面必须更新index.md**：追加一行"链接 + 一句话摘要"
5. **链接格式**：使用Markdown链接 `[text](url)`，不使用裸URL
6. **双向关联**：相关概念之间用文字引用关联（如"参见 [[Agent]] 概念"）

## Ingest流程（摄取新知识）

1. 接收新内容（链接/文件）→ 保存到 `raw/inbox/`
2. 阅读原文 → 提炼原子化知识点
3. 检查wiki中是否已有相关概念页面
4. 不存在则新建，存在则补充新观点
5. 在相关页面之间建立关联
6. 更新 `index.md` 和 `log.md`

## Lint审查（定期维护）

- 找出raw/中有但wiki中未提炼的"未处理资源"
- 找出wiki中没有任何引用的"孤立页面"
- 找出内容高度相似可合并的重复页面
- 标记过时的内容
